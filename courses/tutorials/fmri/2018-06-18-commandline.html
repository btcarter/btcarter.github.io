The Command Line

Jun 18, 2018 • Benjamin Carter

Purpose

This tutorial gives a quick introduction to the command line computing environment.

Learning Outcomes

  1. Be proficient in navigation between directories in the command line environment.
  2. Understand basic shell programming.
  3. Be able to write a for loop to complete a repetitive task.

A Blast from the Past

The command line environment (think old school text based computing from the 70's and 80's or Homestarrunner) predates GUI (graphical user interface) based computing (like Mac OS, Microsoft Windows, and Linux). To the uninitiated it appears colorless, daunting and limited. Luckily for me my first interaction with command line computing happened when I was very young. Our first family computer was an old IBM machine. It had two floppy disk drives (the big ones that were actually floppy), 8-bit color monitor, terrible sound effects, an early version of a Windows file management system (separate from the operating system) and a dot-matrix printer. It resembled Strong Bad's Tandy 400 quite closely. It started in DOS by default and other programs had to be loaded manually. I remember having to switch floppy disks in and out of those disk drives to get some of my favorite games to load. The tedious operations of that early system gave me an appreciation for the borishly simple interface that was the command line as well as for more intuitive GUI based platforms I encountered later in elementary school and used throughout my high school and undergraduate years.

While GUI interfaces have some definite advantages (they're less abstract and more user friendly) they have some distinct failings. Failings the command line interface uses to its advantage. GUI based interfaces are limited to serial processes, i.e. the user must direct each individual process in a step by step iteration. The command line offers the possibility of parallel processing, automation and finer control. It also uses less power than a GUI based system by not devoting as many resources to support to elaborate visual effects. This makes the command line environment appealing developers and scientists. Various programs like Analysis of Functional NeuroImages (AFNI), FSL, Convert3D, and ANTs all operate in the command line environment. Therefore it is important to become familiar with it and attain a basic level of proficiency.

Disclaimer: You Need Unix

Now I need to warn you, from here on out I will be using a Unix based operating system. Windows operating systems generally do not have this capability because they utilize a different kernel (this is the programming that acts as an intermediary between computer hardware and user applications). Luckily you leap this hurdle by installing a Linux subsystem on Windows 10 (which you will have to do to use AFNI anyways). Now is a good time to stop what you're doing and get to work installing this. Do not be surprised if this takes a day or two.

Getting Familiar

Open the command line application on your computer (Command Prompt in Windows, Terminal on Mac and Linux). Let's get oriented to what you are looking at. The first thing you'll notice is that it may not have the same color scheme as mine (you can customize the look and some of the operations via Terminal > Preferences). The next thing you will notice is your blinking cursor (see Figure 2a). This is where you will enter commands to execute. To the left of the cursor you should see an entry called the "path" (see Figure 2b) and displays your current location on your computer, also known as a working directory. Your path will display something like /Users/your/user/name. The region boxed in and labeled as "c" is something you can ignore for the moment and probably won't be present in your terminal window (another customization I've added).

Warning: Do Not Copy & Paste

Copying and pasting commands written by someone else into your terminal robs you of the chance to begin learning how to code. If you really want to understand how the code works you need to go through it line by line and character by character, so you might as well type it out yourself. Don't be lazy. You can do it!

Commanding the Command Line

Very Basic Syntax

Rather than clicking on specific icons to select files or execute tasks the command line requires text input. This text or code has a specific syntax that must be followed. First a command is entered. This is followed by elements called arguments. Commands are specific programs designed to carry out a task. Arguments are input for that program. These are used to modify how a program operates. Arguments include options, file paths, and variables. The basic order of terms looks something like this: command argument1 argument2 argument3 ... . Each program used in the command line has its own rules. You should look them up! Let's look up the help file for the command echo. This can be done within the command line using the command man followed by echo. Type man echo and press enter. You should get output in the terminal that looks something like this:

Press q to exit the man file. The command man (for manual) followed by any other command should bring up a command manual entry for that command. You can use these as a reference when writing code.

Navigation & File Creation

There are two types of objects you will encounter in the Unix environment: files and directories. Files are intuitive - they are a piece of information on the computer (like a text document, Microsoft Excel document, music or an image). Directories are best conceptualized by their common pictographic depiction – a folder. They are used to organize files somewhat like a filing cabinet. They can be organized hierarchically (i.e. folders within folders).

The For Loop

Now that we've covered simple navigation and directory creation we can move onto a more complex topic: the For Loop. This command is used to carryout repetitive processes so the user does not have to manually complete initiate every single iteration. Ever had to edit a but of file names so they all have consistent labeling, or make repetitive changes to a CSV file in Excel? Wish there were a faster way to do that? For loops are your new best friend. For loops are especially useful in neuroimaging during image preprocessing and analysis, enabling the researcher to give a generic set of commands to be carried out on each participant's dataset and then execute these as a group rather than manually do it for each subject.

Don't worry, it gets better

This is just a small taste of the number of commands and processes that are available in the command line. We are going to use and install quite a few more! It may seem obtuse at first but once you master the command line a whole new world of efficiency will be opened up to you. If this is your first time venturing beyond GUI interfaces then take this opportunity to stretch yourself a bit. This is how your parents and grandparents were introduced to computers! If all of this seems a bit much to wrap your head around at the moment, don't worry too much about it. Help is out there! Computer coding is one of the few egalitarian pursuits left in the world. A host of online services exist offering free tutorials. Get involved in a web forum. With the right search term or question you should be able to find an answer.
