Working with Windows

Now I've spent the last six years using Mac and Linux machines to do most of my computing. This has mostly been driven by the fact that many of the scientific computing packages I use are written and developed on Unix. The reality of the situation though is that most of the personal computers in the world, especially those used by students, run some version of Microsoft, which until recently could not natively run any Unix software. Microsoft has now implemented a Unix based kernel, enabling users to install and run Ubuntu from the app Microsoft app store. Took them long enough, especially if you consider that Mr. Gates originally wanted to use Unix back when he was developing Microsoft in the 70's and 80's.

My historical griping aside (tends to happen a lot, just ask my family), the Unix kernel should hopefully rectify the problem, but there will probably be some turbulence. I'll try and smooth that out as we go. In the mean time there are a few things you need to do to make your personal set up as close to mine as you can get it.

Install Ubuntu

You will need the most recent version of the Microsoft OS (Windows 10 or better). Once you have that you need to do the following to install Ubuntu:

  1. Turn on Developer Mode: Open settings,choose Update & Security, choose For Developers, activate Developer Mode.
  2. Turn on Kernel: Open Control Panel, select Turn Windows features on or off, scroll to the bottom of the window that pops up and check the Windows subsystem for Linux. Click Ok. Restart your computer.
  3. Install Ubuntu: Open the Windows store, search for Ubuntu (has an orange icon). Install it.
  4. Create a user name and password: Use whatever you want, but don't forget it. The cursor won't move while you type your password but your keystrokes are being recorded. If you make a mistake then just hold down backspace for a bit and you should be good to go.

Setting up your profile

Now we can set some basic environmental defaults to ease computing in Ubuntu and bash. These can be set by creating a file called .bash_profile. Adding a period in front of the file name makes it a hidden file so it will not appear in the directory when you use the ls command, which helps ensure you don't delete or modify it by accident. This file should be created in your home directory (the default directory that Ubuntu opens with).

Lets make some edits to the file by inserting the following lines. Where you see Windows Profile Name please insert your personal Windows Profile Name. If you don't know what that is just use the command ls //mnt/c/Users/. This should print a list to your screen of every user profile on your computer. Yours should be among them.

To save these changes press Control + O. Confirm the file name, and then press Control + X to exit nano. Now enter source ~/.bash_profile to activate these new settings in your terminal.

Now to explain what we just did. Every time you sart the Ubuntu terminal, Linux looks for this file by default to set up the environment. Those two lines we entered create two custom commands you can use to help you navigate around your computer. The first (alias hm='//mnt/c/Users//') created a command you can use to navigate into your Windows profile, saving you having to cd your way into it each time you start Ubuntu. Now all you need to do is type hm instead.

The other command (cl) is a command that I created to satisfy my compulsive need for a clean screen. (Seriously it's bad. I don't even have a wallpaper for my desktop right now to avoid visual clutter, it's just black.) This combines the three commands clear, pwd, and ls. I use it whenever I need a clean terminal plus a reminder of where I'm currently working.

You can create aliases for just about anything. Just be careful and always remember where .bash_profile is in case you need to modify it later.

The Home Directory

The Home directory is the directory your terminal window opens to by default (just type echo $HOME to see what it is currently set to). Your home directory for your linux profile is set to /home/user name right now. This can be an inconvenient place to start since you may actually store most of your files in a different location (this is why I suggest creating the alias hm to give you a shortcut to where you actually want to start working without having to actually change your home directory. I have a sneaking suspicion doing that could make things messy later).
